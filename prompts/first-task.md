# 초기 인수용 첫 작업 프롬프트

## 사용 범위

이 문서는 **일반 Codex 작업의 기본 브리프가 아니다**.
아래처럼 제한된 경우에만 참고한다.

- 저장소를 처음 인수하는 초기 부트스트랩 작업
- 구조 파악이 먼저 필요한 대규모 첫 점검
- 과거 작업 의도와 초기 우선순위를 확인해야 하는 아카이브 참고

일반 작업에서는 이 문서를 자동으로 적용하지 않는다.
일반 작업의 기본 우선순위는 아래와 같다.

1. 현재 사용자 요청
2. 루트 `AGENTS.md`
3. 하위 디렉터리 `AGENTS.md`
4. 필요한 경우에만 보조 프롬프트 문서

## 문서 성격

이 문서는 저장소가 아직 구조화되지 않았거나, 첫 인수자가 전체 흐름을 빠르게 파악해야 할 때 쓰는 **초기 가이드**다.
현재 운영 단계에서는 루트 `AGENTS.md`가 더 우선한다.

## 보관된 초기 부트스트랩 요약

이 저장소는 장기적으로 멀티 에이전트 자동매매 플랫폼으로 확장될 수 있지만,
초기 인수 단계의 핵심 목표는 아래와 같았다.

- Binance Futures 실거래 코어 구조 파악
- AI 판단, 리스크 검증, 주문 실행, 상태 관리의 경계 분리
- 실거래 안전성 중심 리팩터링
- 감사 가능성과 운영 가시성 확보

초기 인수자가 참고할 때도 아래 원칙은 유지한다.

- 전면 재작성보다 현재 구현 적응 우선
- AI보다 결정론적 정책 우선
- 실주문은 승인, 리스크 검증, 실행 통제를 거쳐야 함
- pause, emergency, degraded, audit trail 유지
- 상태가 불확실하면 신규 진입보다 보호와 차단을 우선

## 현재 단계에서의 해석 주의

이 문서에 적힌 광범위한 구현 목록을 일반 작업마다 그대로 따르지 않는다.
현재는 **작은 범위의 안전한 수정**이 기본이며, 큰 작업은 별도 분석과 단계 분리가 선행되어야 한다.

즉, 이 문서는 다음 용도에 가깝다.

- 왜 이 저장소가 실거래 안전 구조를 우선하게 되었는지 설명
- 초기에 어떤 경계를 중요하게 봤는지 기록
- 후속 인수자가 장기 맥락을 이해하도록 돕는 참고 자료
