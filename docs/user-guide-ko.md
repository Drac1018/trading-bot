# 사용자 가이드

## 문서 목적

이 문서는 `Multi-Agent Trading MVP`를 실제로 사용하는 운영자 기준 안내서입니다.
개발이나 코드 수정이 아니라, 화면 접속부터 설정, 실거래 승인, 로그 확인, 문제 확인까지 사용자 관점으로 설명합니다.

## 이 프로그램이 하는 일

이 프로그램은 Binance 선물 계정과 OpenAI를 연결해 다음 흐름으로 동작합니다.

1. Binance에서 시장 데이터와 계정 상태를 가져옵니다.
2. AI가 거래 판단을 생성합니다.
3. 리스크 엔진이 판단을 다시 검사합니다.
4. 승인 조건을 모두 만족하면 실거래 주문을 실행합니다.
5. 주문, 체결, 리스크, 감사 로그를 대시보드에 기록합니다.

중요한 점:

- AI가 바로 주문을 넣는 구조가 아닙니다.
- 리스크 엔진과 실거래 가드가 항상 우선합니다.
- 실거래는 `환경 게이트 + 실거래 사용 + 승인 창 + 거래중지 해제`가 모두 맞아야 가능합니다.

## 접속 주소

- 프런트 대시보드: `http://127.0.0.1:3000`
- 백엔드 상태 확인: `http://127.0.0.1:8000/health`

## 첫 사용 순서

### 1. 메인 화면 접속

브라우저에서 `http://127.0.0.1:3000` 으로 접속합니다.

메인 화면에서는 아래를 먼저 확인합니다.

- 현재 운영 모드
- Binance / OpenAI 연결 상태
- 실거래 가능 여부
- 최근 차단 사유
- 최근 주문/포지션 상태

### 2. 설정 페이지로 이동

설정 화면:

- `http://127.0.0.1:3000/dashboard/settings`

이 페이지에서 실제 운용의 대부분을 제어합니다.

## 설정 화면 사용법

설정 화면은 크게 네 가지를 담당합니다.

1. 실거래 가드
2. Binance 연결
3. OpenAI 연결
4. 추적 심볼 설정

### 1. Binance 연결

입력 항목:

- `Binance API Key`
- `Binance API Secret`
- `Binance 시세 사용`
- `USD-M Futures`
- 필요 시 `Testnet 사용`

사용 방법:

1. Binance Key와 Secret를 입력합니다.
2. `설정 저장`을 누릅니다.
3. `Binance 연결 확인` 버튼으로 정상 연결 여부를 확인합니다.

정상이라면 아래를 확인할 수 있습니다.

- 계정 잔고
- 사용 가능 잔고
- 오픈 포지션
- 미체결 주문

### 2. OpenAI 연결

입력 항목:

- `OpenAI API Key`
- AI 사용 여부
- AI 모델
- AI 최소 호출 간격

사용 방법:

1. OpenAI Key를 입력합니다.
2. `설정 저장`을 누릅니다.
3. `AI 연결 확인` 또는 관련 상태 카드에서 설정 여부를 확인합니다.

주의:

- `AI 사용`이 꺼져 있으면 시장 데이터 수집만 하고, AI 의사결정은 돌지 않습니다.
- AI가 꺼져 있으면 의사결정/리스크/에이전트 실행 기록도 자동 생성되지 않도록 설계되어 있습니다.

### 3. 실거래 가드

실거래는 아래 4가지가 동시에 맞아야 합니다.

1. `실거래 사용` 켜짐
2. `거래 일시중지` 꺼짐
3. `실거래 승인` 창 열림
4. 환경 게이트 활성화

화면에 보이는 운영 모드 의미:

- `거래 중지`: 전역 중지 상태
- `가드 모드`: 연결은 되어 있지만 아직 거래 가능 상태는 아님
- `실행 가능`: 실거래 조건이 모두 열려 있음

버튼 의미:

- `즉시 중지`: 전역 거래를 멈춥니다.
- `중지 해제`: 전역 거래중지를 해제합니다.
- `실거래 승인`: 일정 시간 동안 실거래 승인 창을 엽니다.
- `승인 해제`: 열린 승인 창을 닫습니다.
- `거래소 동기화`: Binance 계정 상태를 다시 읽어옵니다.
- `실주문 사전 점검`: 현재 심볼 기준으로 주문 조건을 점검합니다.

### 4. 심볼 선택

설정 화면에서 추적할 심볼을 고를 수 있습니다.

- 빠른 선택 버튼으로 선택
- 추가 심볼 직접 입력 가능
- 여러 심볼을 동시에 추적 가능

예:

- `BTCUSDT`
- `ETHUSDT`
- `SOLUSDT`

심볼을 여러 개 선택하면 AI, 리스크, 주문 로그도 심볼별로 쌓입니다.

## 계정 페이지 사용법

계정 화면:

- `http://127.0.0.1:3000/dashboard/account`

이 페이지에서는 실제 Binance 계정 상태를 봅니다.

확인 가능한 항목:

- 총 지갑 잔고
- 사용 가능 잔고
- 총 마진 잔고
- 오픈 포지션
- 미체결 주문
- 자산별 잔고

이 화면은 실제 Binance 응답을 기준으로 표시됩니다.
즉, 포지션 수량이나 잔고는 로컬 더미 값이 아니라 연결된 실계좌 값입니다.

## 주요 페이지 설명

### 1. 시장 / 신호 스냅샷

- 주소: `http://127.0.0.1:3000/dashboard/market`

보는 것:

- 최신 시장 스냅샷
- 가격/거래량
- 계산된 피처
- 데이터가 오래됐는지 여부

언제 확인하나:

- AI가 왜 그런 판단을 했는지 보고 싶을 때
- 시장 데이터가 최신인지 점검할 때

### 2. 거래 의사결정

- 주소: `http://127.0.0.1:3000/dashboard/decisions`

보는 것:

- AI의 `hold / long / short / reduce / exit`
- 신뢰도
- 손절가 / 익절가
- 설명과 근거 코드
- AI 공급자

주의:

- 여기서 AI가 `long`이라고 해도 바로 주문되는 것이 아닙니다.
- 반드시 리스크 페이지 결과를 같이 확인해야 합니다.

### 3. 리스크 상태

- 주소: `http://127.0.0.1:3000/dashboard/risk`

보는 것:

- 허용 / 차단 여부
- 차단 사유 코드
- 레버리지/리스크 비중
- 알림

자주 보게 되는 차단 사유 예:

- `TRADING_PAUSED`
- `HOLD_DECISION`
- `LIVE_ENV_DISABLED`
- `STALE_MARKET_DATA`
- `MISSING_STOP_OR_TARGET`

이 페이지는 “왜 거래가 안 나갔는지”를 확인하는 1순위 화면입니다.

### 4. 실거래 주문 / 체결

- 주소: `http://127.0.0.1:3000/dashboard/orders`

보는 것:

- 실거래 주문 상태
- 요청 수량
- 실제 체결 수량
- 평균 체결가
- 보호 주문 관계
- 거절 사유

부분 체결, 거절, 보호 주문 연결까지 확인할 수 있습니다.

### 5. 실거래 포지션

- 주소: `http://127.0.0.1:3000/dashboard/positions`

보는 것:

- 현재 보유 포지션
- 진입가
- 현재가
- 손익
- 레버리지
- 손절가 / 익절가

### 6. 에이전트 실행

- 주소: `http://127.0.0.1:3000/dashboard/agents`

보는 것:

- 어떤 AI 역할이 실행됐는지
- 어떤 공급자를 썼는지
- 입력/출력 요약
- 성공/실패 여부

현재 운영 범위의 AI 역할:

- 거래 의사결정 AI
- 최종 리뷰 AI

### 7. 스케줄러

- 주소: `http://127.0.0.1:3000/dashboard/scheduler`

보는 것:

- 15분 의사결정 주기
- 1h / 4h / 12h / 24h 리뷰 실행
- 최근 실행 결과
- 다음 실행 시각

최신 데이터가 안 바뀌면 이 페이지와 서비스 상태를 먼저 확인하세요.

### 8. 감사 로그

- 주소: `http://127.0.0.1:3000/dashboard/audit`

보는 것:

- 설정 변경
- 실거래 승인/해제
- 주문 실행/거절
- 동기화
- 에이전트 실행
- 스케줄러 실행

운영 이슈가 생기면 가장 먼저 이 화면으로 원인을 추적합니다.

## 실제 운용 체크리스트

실거래를 시작하기 전에 아래를 꼭 확인합니다.

1. Binance 연결 성공
2. OpenAI 연결 성공
3. `AI 사용` 켜짐
4. `실거래 사용` 켜짐
5. `거래 일시중지` 꺼짐
6. `실거래 승인` 창 열림
7. 추적 심볼 확인
8. 계정 페이지에서 사용 가능 잔고 확인
9. 주문 / 체결 화면이 비정상 에러 없이 열리는지 확인

## 운영 모드가 예상과 다를 때

### 1. `거래 중지`로 보일 때

확인할 것:

- 설정 화면에서 `거래 일시중지`가 켜져 있는지
- `즉시 중지`를 눌러둔 상태가 아닌지

해결:

- `중지 해제`를 누릅니다.

### 2. `가드 모드`로 보일 때

확인할 것:

- 실거래 사용이 켜져 있는지
- 실거래 승인 창이 열려 있는지
- 환경 게이트가 활성화되어 있는지
- Binance Key/Secret이 저장돼 있는지

해결:

- 설정 저장
- 실거래 승인
- 필요한 경우 관리자에게 환경 게이트 설정 확인

### 3. 데이터가 최신화되지 않을 때

확인할 것:

- 스케줄러 서비스가 살아 있는지
- 시장 스냅샷 시간이 오래된 건 아닌지
- 설정 화면에서 AI가 꺼져 있지는 않은지

해결:

- 스케줄러 재시작
- 감사 로그와 스케줄러 페이지 확인

### 4. AI가 계속 `결정론적 Mock`으로 보일 때

의미:

- OpenAI 호출이 실패해 fallback 되었을 가능성이 큽니다.

확인할 것:

- OpenAI Key 저장 여부
- 최근 실패 사유
- 설정 화면의 AI 사용량 카드

## 자주 하는 질문

### Q1. 실거래 승인을 눌렀는데 바로 주문이 안 나갑니다.

가능한 원인:

- 현재 판단이 `hold`
- 리스크 엔진이 차단
- 승인 창은 열렸지만 거래중지 상태
- 시장 데이터가 오래됨

### Q2. 주문이 거절됐습니다.

가능한 원인:

- 증거금 부족
- 최소 주문 금액 미달
- 레버리지/리스크 제한
- Binance 필터 조건 불일치

주문 / 체결 화면과 감사 로그를 같이 확인하세요.

### Q3. 잔고 기준은 무엇을 사용하나요?

현재 실거래 경로에서는 로컬 시작 자금이 아니라 Binance 실계좌의 잔고/마진 값을 기준으로 동작합니다.

### Q4. AI를 꺼도 시장 데이터는 보이나요?

네. AI를 꺼도 시장 데이터 수집은 유지될 수 있습니다.
다만 AI 의사결정과 관련 실행 기록은 생성되지 않습니다.

## 권장 운영 습관

- 실거래 전에는 항상 계정 페이지에서 사용 가능 잔고 확인
- 실거래 승인 직후에는 주문/체결 화면과 감사 로그 함께 확인
- 이상 징후가 있으면 먼저 `즉시 중지`
- 원인 분석은 `리스크 -> 주문/체결 -> 감사 로그` 순서로 확인
- 설정 변경 후에는 저장과 연결 확인을 모두 수행
